# GitHub Branch Kya Hai?

**Branch** ka matlab hai ek alag rasta ya copy. Jab aap kisi project par kaam karte hain aur koi naya feature banana chahte hain bina main code ko kharab kiye, tab aap ek nayi branch banate hain.

Socho ek ped (tree) hai. Uska main tana (trunk) aapka main code hai. Branches us ped ki shakhayein hain jahan aap naye experiments kar sakte hain. Jab experiment successful ho jaye, toh aap branch ko wapas main tane mein jod (merge) dete hain.

## Branch Kyun Zaroori Hai?
- **Safety:** Main code hamesha safe aur working condition mein rehta hai.
- **Teamwork:** Ek hi time par alag-alag developers alag-alag branches par kaam kar sakte hain.

> [!TIP]
> `main` ya `master` branch hamesha production-ready code ke liye use hoti hai. Hamesha naye features ke liye nayi branch banayein!
